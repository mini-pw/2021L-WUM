Nauka modeli i ocena jakości¶

Stworzymy trzy rne modele klasyfikacyjne. Kady z nich bdziemy sprawdzali, wykorzystujc trzy miary jakoci:

  • accuracy,
  • F1 score (rednia harmoniczna z precission i recall)
  • ROC AUC (czy recall i False Positive Rate (1 - specificity)). Dziki zastosowaniu rnych miar bdziemy mieli wiksz pewno odnonie jakoci wytrenowanych klasyfikatorw, gdy intuicyjne accuracy moe go nie odzwierciedla. Daj nam rwnie dodatkowe informacje o popenianych bdach - np. precyzji modelu i uciekajcych predykcjach (recall).

Wyliczymy rednie wartoci poszczeglnych miar w kroswalidacji na zbiorze treningowym oraz wartoci dla zbioru testowego.

# Funkcja pomocnicza agregujca miary z kroswalidacji i tworzca DataFrame do pniejszej wizualizacji
def get_results(cv_results, model_name):
    print("--- Performance measures values for", model_name, "---")
    for measure in ["test_accuracy", "test_f1", "test_roc_auc"]:
        print(measure.split("_", 1)[1], " | ",  f"mean: {np.mean(cv_results[measure]):.4f}, min: {min(cv_results[measure]):.4f}, max: {max(lr_cv_results[measure]):.4f}")
    return pd.DataFrame({"model": model_name, "accuracy_score": cv_results["test_accuracy"], "f1_score": cv_results["test_f1"], "roc_auc_score": cv_results["test_roc_auc"]})

Pierwszy model - regresja logistyczna¶

Wybrane hiperparametry:

  • penalty = 'lasticnet' - regresja liniowo aczca kary Ridge ($L_2$) i Lasso($L_1$) ograniczajce wartoci wag,
  • solver = 'saga' - algorytm uyty w optymalizacji wag, konieczny do zastosowania ElasticNet,
  • l1_ratio = 0.8 - wspczynnik istotnoci kary $L_1$ w branej pod uwag kombinacji liniowej.
In [11]:
from sklearn.linear_model import LogisticRegression

lr = LogisticRegression(penalty='elasticnet', solver='saga', l1_ratio=0.8)

lr_model = Pipeline(steps=[('standardscaler', StandardScaler()),   #stosujemy standaryzacj 
                ('lr', lr)])  

lr_cv_results = cross_validate(lr_model, 
                 X_train, y_train, 
                 cv=split,
                 scoring=["accuracy", "f1", "roc_auc"],
                          n_jobs=-1)

lr_res_df = get_results(lr_cv_results, "LogisticRegression")

Drugi model - las losowy¶

Wybrane hiperparametry:

  • min_samples_leaf = 3 - minimala liczba obserwacji, ktre musz znale si w liciu drzewa - wiksze wartoci zapobiegaj duej wariancji,
  • min_samples_split = 7 - minimalna liczba obserwacji w wle konieczna do podzielenia go - podobnie jak wyej,
  • max_depth = 20 - maksymalna gboko pojedynczego drzewa - zmniejszenie powoduje zmniejszenie wariancji,
  • n_estimators = 200 - liczba drzew w modelu,
  • max_features = 'sqrt' - maksymalna liczba cech rozwaanych przy dzieleniu wzw w drzewach.
In [12]:

Trzeci model - klasyfikator metodą k najbliższych sąsiadów¶

Wybrane hiperparametry:

  • n_neighbors = 8 - liczba ssiadw branych pod uwag w tworzeniu predykcji,
  • weights = 'distance' - waga poszczeglnych ssiadw w predykcji zwizana jest z odlegoci od obserwacji - czym mniejsza, tym wiksza waga,
  • p = 1.5 - metryka odlegoci (w tym przypadku uoglniona odlego Minkowskiego $L_{1.5}$).

from sklearn.metrics import accuracy_score, f1_score, roc_auc_score

test_results = []
for model, model_name in [(lr_model, "LogisticRegression"), (rf_model, "RandomForest"), (knn_model, "KNN")]:
    model.fit(X_train, y_train)
    y_pred = model.predict(X_test)
    measures_results = {"model": model_name}
    for measure in [accuracy_score, f1_score, roc_auc_score]:
        measures_results[measure.__name__] = measure(y_test, y_pred)
    test_results.append(measures_results)
    
test_results_df = pd.DataFrame(test_results)

Wnioski¶

  • Najlepszym z wytrenowanych modeli okazuje si by las losowy, dla ktrego kada z rozwaanych miar jest lepsza od ich wartoci dla innych modeli.
  • Nie mona si jednak przywizywa do wynikw modeli, gdy zostay one wytrenowane dla konkretnych wartoci hiperparametrw, ktre nie zostay zoptymalizowane.
  • W zalenoci od tego, na czym nam zaley moglibymy optymalizowa model pod ktem rnych miar:
    • moemy przykadowo chcie przewidywa kady dzie deszczowy (optymalizowa recall) albo nie chcie tworzy faszywych predykcji i przewidywa deszczu, gdy go nie bdzie (optymalizowa precission) lub w poczeniu rozwaa F1 score, co wydaje sinajbardziej sensowne.
  • Wybr miary klasyfikatora nie jest atwy. Na pewno warto rozpatrze wicej miar ni accuracy (zauwamy bowiem, e np. w przypadku niezbalansowanych klas moe on dawa wysokie wyniki, przewidujc zawsze bardziej liczn klas).
